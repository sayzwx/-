# 编译原理核心术语详解

## 一、基本概念

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **文法 (Grammar)** | Grammar | 形式化描述语言结构的规则系统，由终结符、非终结符、产生式和开始符号组成 | 是所有分析方法的理论基础 |
| **终结符 (Terminal)** | Terminal Symbol | 构成语言基本符号的不可再分的元素，如关键字、运算符、标识符 | 是句型中最终出现的符号 |
| **非终结符 (Non-terminal)** | Non-terminal Symbol | 表示语法结构的语法变量，可被进一步展开 | 通过产生式推导为终结符串 |
| **产生式 (Production)** | Production Rule | 形如 A → α 的规则，定义非终结符如何推导 | 文法的核心组成部分 |
| **开始符号 (Start Symbol)** | Start Symbol | 文法中特殊的非终结符 S，所有合法句子都从它推导出来 | 推导的起点 |

## 二、推导与句型

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **推导 (Derivation)** | Derivation | 从开始符号出发，反复应用产生式替换非终结符，得到句子的过程 | 分为最左推导和最右推导 |
| **最左推导 (Leftmost Derivation)** | Leftmost Derivation | 每步都替换最左边的非终结符 | 与自顶向下分析对应 |
| **最右推导 (Rightmost Derivation)** | Rightmost Derivation | 每步都替换最右边的非终结符，又称规范推导 | 与自底向上分析（规范规约）对应 |
| **句型 (Sentential Form)** | Sentential Form | 从开始符号经过若干步推导得到的符号串（可含非终结符） | 句子是只含终结符的特殊句型 |
| **句子 (Sentence)** | Sentence | 只含终结符的句型，是语言中的合法串 | 是推导的最终结果 |
| **短语 (Phrase)** | Phrase | 句型中由某个非终结符推导出的子串。若 S ⇒* αAδ 且 A ⇒+ β，则 β 是句型 αβδ 中相对于 A 的短语 | 是规范规约中识别归约对象的基础 |
| **直接短语 (Immediate Phrase)** | Simple Phrase | 一步推导得到的短语，即 A → β 中 β 是相对于 A 的直接短语 | 句柄是最左直接短语 |
| **句柄 (Handle)** | Handle | 句型的最左直接短语，是自底向上分析中当前应被归约的子串 | 规范归约的核心概念，LR 分析器寻找句柄 |
| **规范句型 (Canonical Form)** | Canonical Sentential Form | 由最右推导得到的句型 | 规范规约的操作对象 |

## 三、语法分析方法总览

| 术语 | 英文 | 含义 | 代表算法 | 与其他概念的联系 |
|------|------|------|---------|-----------------|
| **自顶向下分析 (Top-down)** | Top-down Parsing | 从开始符号出发，逐步推导出输入串（构造语法树从根到叶） | LL(1)、递归下降、预测分析 | 对应最左推导；需要消除左递归和提取左公因子 |
| **自底向上分析 (Bottom-up)** | Bottom-up Parsing | 从输入串出发，逐步归约到开始符号（构造语法树从叶到根） | LR(0)、SLR(1)、LALR(1)、LR(1)、算符优先 | 对应最右推导的逆过程（规范规约） |
| **预测分析 (Predictive Parsing)** | Predictive Parsing | 根据向前看符号（lookahead）选择产生式，无需回溯 | LL(1) 分析表 | 是自顶向下分析的高效实现方式 |
| **递归下降 (Recursive Descent)** | Recursive Descent | 每个非终结符对应一个递归函数来实现分析 | 手写编译器常用 | 是预测分析的一种实现形式 |

## 四、LL 分析相关

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **LL(k) 文法** | LL(k) Grammar | 从左到右扫描、最左推导、向前看 k 个符号即可唯一确定产生式的文法 | LL(1) 是最常用的情况 |
| **FIRST 集** | FIRST Set | 从某个符号串能推导出的所有可能的第一个终结符的集合 | 用于构造 LL(1) 分析表 |
| **FOLLOW 集** | FOLLOW Set | 在某个句型中，紧跟在某个非终结符之后可能出现的所有终结符的集合 | 与 FIRST 集一起构造 LL(1) 预测分析表 |
| **LL(1) 分析表 (Parsing Table)** | LL(1) Parse Table | 二维表，行是非终结符，列是终结符，表项是应选的产生式或报错 | 是表驱动的自顶向下分析的核心 |
| **左递归 (Left Recursion)** | Left Recursion | 形如 A → Aα 的产生式，会导致自顶向下分析无限循环 | 必须消除后才能使用 LL 分析 |
| **左公因子 (Left Factoring)** | Left Common Factor | 多个产生式有相同前缀（A → αβ₁ \| αβ₂），导致无法预测选择 | 提取左公因子后使文法满足 LL(1) 条件 |

## 五、LR 分析相关

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **LR(k) 文法** | LR(k) Grammar | 从左到右扫描、最右推导的逆过程、向前看 k 个符号即可唯一确定归约的文法 | LR(1) 能分析的文法类是 LL(1) 的超集 |
| **LR(0) 项目 (Item)** | LR(0) Item | 产生式中带一个圆点标记分析进度，如 A → α·β | 是构造 LR 分析状态的基础 |
| **LR(1) 项目** | LR(1) Item | LR(0) 项目加上向前看符号，如 [A → α·β, a] | 提供精确的向前看信息，避免冲突 |
| **项目集规范族 (Canonical Collection)** | Canonical Collection of Item Sets | 所有有效项目集的集合，每个项目集对应分析表的一个状态 | 是构造 LR 分析表的中间产物 |
| **GO 函数 (GOTO)** | GOTO Function | GOTO(I, X) 表示项目集 I 经符号 X 转移后得到的新项目集 | 构建分析状态之间的转移关系 |
| **CLOSURE 函数** | CLOSURE Function | 对项目集中圆点后的非终结符，将其所有产生式的初始项目加入集合 | 计算完整的项目集 |
| **SLR(1)** | Simple LR(1) | 用 FOLLOW 集来解决 LR(0) 中的冲突，比 LR(0) 强，比 LR(1) 弱 | 是 LR(0) 和 LR(1) 的折中 |
| **LALR(1)** | Lookahead LR(1) | 合并 LR(1) 项目集中同心（核心相同）的项目集以减少状态数 | Yacc/Bison 默认使用；能力介于 SLR(1) 和 LR(1) 之间 |

## 六、规范规约

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **规范规约 (Canonical Reduction)** | Canonical Reduction | 最右推导的逆过程，每步归约当前句型的最左句柄 | 是自底向上分析的正确归约策略 |
| **规范归约的步骤** | — | 1. 移入输入符号到栈中；2. 识别栈顶的句柄；3. 用产生式左部替换句柄 | LR 分析器通过状态栈自动完成此过程 |
| **活前缀 (Viable Prefix)** | Viable Prefix | 规范句型的前缀，该前缀不会越过句柄的右端 | LR 分析器栈中保存的内容就是活前缀 |

## 七、算符优先分析

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **算符优先文法 (Operator Precedence Grammar)** | OPG | 不含 ε 产生式，且任何两个终结符之间至多有一种优先关系的文法 | 是一种特殊的自底向上分析方法 |
| **算符优先关系 (Precedence Relation)** | Operator Precedence Relation | 终结符之间的三种关系：<（低于）、=（等于）、>（高于） | 用于确定归约的边界 |
| **优先关系表 (Precedence Table)** | Precedence Table | 存储所有终结符对之间优先关系的二维表 | 类似于 LR 分析表，但更简单 |
| **优先函数 (Precedence Function)** | Precedence Function | 用两个函数 f 和 g 代替优先关系表，将关系比较转化为数值比较 | 节省存储空间，f(a) < g(b) 等价于 a < b |
| **最左素短语 (Leftmost Prime Phrase)** | Leftmost Prime Phrase | 至少含一个终结符且不含更小的含终结符短语的短语 | 算符优先分析中的归约对象（不是句柄） |

### 算符优先 vs 规范规约

| 对比维度 | 算符优先分析 | 规范规约 (LR) |
|---------|------------|--------------|
| 归约对象 | 最左素短语 | 句柄 |
| 分析能力 | 较弱（仅 OPG 文法） | 强（LR 文法类） |
| 实现复杂度 | 简单 | 较复杂 |
| 错误检测 | 较弱 | 较强 |
| 典型应用 | 表达式解析 | 通用语法分析 |

## 八、表驱动分析

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **表驱动分析 (Table-driven Parsing)** | Table-driven Parsing | 使用预构造的分析表驱动分析动作（移入/归约/接受/报错），而非硬编码逻辑 | LL(1) 和 LR 系列都采用此模式 |
| **分析表 (Parse Table)** | Parse Table | 以当前状态和输入符号为索引，指示应执行的分析动作 | LR 分析表含 ACTION 和 GOTO 两部分 |
| **ACTION 表** | ACTION Table | LR 分析表的一部分，指示在当前状态和终结符下的动作（移入/归约/接受/报错） | 与 GOTO 表配合使用 |
| **GOTO 表** | GOTO Table | LR 分析表的一部分，指示在当前状态和非终结符下转移到哪个新状态 | 归约后使用 |
| **移入-归约冲突 (Shift-Reduce Conflict)** | Shift-Reduce Conflict | 在某个状态下，分析表同时指示移入和归约 | 说明文法不是该文法类（如不是 SLR），需要改写文法 |
| **归约-归约冲突 (Reduce-Reduce Conflict)** | Reduce-Reduce Conflict | 在某个状态下，分析表对同一输入有多个归约选择 | 文法有二义性或不适合该分析技术 |

## 九、冲突与二义性

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **二义文法 (Ambiguous Grammar)** | Ambiguous Grammar | 同一个句子存在多棵不同的语法树（多个最左推导） | 二义文法不是 LR 文法；可通过优先级/结合性规则解决 |
| **悬空 else (Dangling Else)** | Dangling Else | if-else 语句的经典二义性问题 | 通常通过"最近匹配"规则消除 |
| **优先级 (Precedence)** | Precedence | 运算符执行顺序的约定 | 可通过声明优先级消除移入-归约冲突 |
| **结合性 (Associativity)** | Associativity | 相同优先级运算符的结合方向（左结合/右结合） | 辅助消除冲突 |

## 十、文法类层次关系

```
文法能力（从弱到强）：

  LL(1)  ⊂  SLR(1)  ⊂  LALR(1)  ⊂  LR(1)

  算符优先文法 (OPG) 与上述系列有交集但不完全包含
```

| 文法类 | 分析方向 | 分析能力 | 分析表大小 | 实际应用 |
|-------|---------|---------|-----------|---------|
| LL(1) | 自顶向下 | 最弱 | 小 | 手写递归下降编译器 |
| SLR(1) | 自底向上 | 较强 | 中 | 教学示例 |
| LALR(1) | 自底向上 | 强 | 中 | Yacc / Bison / CUP |
| LR(1) | 自底向上 | 最强 | 大 | 理论研究，状态数过多 |
| OPG | 自底向上 | 特殊 | 小 | 表达式子语言 |

## 十一、中间代码与语义分析（扩展）

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **属性文法 (Attribute Grammar)** | Attribute Grammar | 在上下文无关文法基础上附加语义属性和语义规则 | 语法分析的延伸，用于语义检查 |
| **S-属性定义 (S-attributed)** | S-attributed Definition | 只含综合属性的属性文法 | 可在自底向上分析中计算 |
| **L-属性定义 (L-attributed)** | L-attributed Definition | 属性依赖限于从左到右计算的属性文法 | 可在自顶向下分析中计算 |
| **语法制导翻译 (Syntax-Directed Translation)** | SDT | 在语法分析过程中嵌入语义动作 | 连接语法分析与中间代码生成 |
| **三地址码 (Three-Address Code)** | TAC | 中间代码形式，每条指令最多三个操作数 | 语法分析之后的中间表示 |
| **抽象语法树 (AST)** | Abstract Syntax Tree | 源代码语法结构的树形抽象表示 | 语法分析的输出，语义分析的输入 |

## 十二、短语、直接短语、句柄的求法

**基本方法：画出语法树，根据树的结构判定。**

以文法 `S → aAB`，`A → Bb | c`，`B → d` 为例，输入串 `adb`：

```
        S
      / | \
     a  A  B
        |
        B
        |
        b（注：此处为示例说明结构）

实际语法树：
        S
      / | \
     a  A  B
        |  |
        B  d
        |
        b

（以实际推导为准）
```

**规则总结：**

| 概念 | 判定方法 |
|------|---------|
| **短语** | 语法树中每个内部节点（非终结符）对应的子树叶节点串，都是相对于该非终结符的短语 |
| **直接短语** | 只有父子两层的子树（即一步推导 A → β）对应的叶节点串 |
| **句柄** | 最左直接短语（在语法树上最靠左的两层子树） |

**通用步骤：**
1. 构造输入串的语法树
2. 找出所有子树 → 每个子树的叶节点串即为短语
3. 找出只有两层的子树 → 叶节点串即为直接短语
4. 在所有直接短语中取最左边的 → 即为句柄

## 十三、FIRST 集的求法

**定义：** FIRST(α) = { α 能推导出的所有以终结符开头的串的第一个终结符 }，若 α ⇒* ε，则 ε ∈ FIRST(α)。

**计算规则：**

设 `X` 是文法符号（终结符或非终结符），`α = X₁X₂...Xₙ`：

| 情况 | 规则 |
|------|------|
| `X` 是终结符 | FIRST(X) = { X } |
| `X` 是非终结符，且有 `X → ε` | 将 ε 加入 FIRST(X) |
| `X` 是非终结符，且有 `X → a...` | 将 a 加入 FIRST(X) |
| `X → Y₁Y₂...Yₖ`（Yᵢ 是非终结符） | 将 FIRST(Y₁) 中非 ε 元素加入 FIRST(X)；若 ε ∈ FIRST(Y₁)，则继续看 FIRST(Y₂)，以此类推；若所有 Y₁~Yₖ 都能推导出 ε，则将 ε 加入 FIRST(X) |

**对于串 `X₁X₂...Xₙ` 的 FIRST 集：**
1. 将 FIRST(X₁) 的所有非 ε 元素加入
2. 若 ε ∈ FIRST(X₁)，则将 FIRST(X₂) 的所有非 ε 元素加入
3. 依此类推，若所有 X₁~Xₙ 都含 ε，则 ε 属于 FIRST(X₁X₂...Xₙ)

**示例：**

文法：
```
E → TE'
E' → +TE' | ε
T → FT'
T' → *FT' | ε
F → (E) | id
```

计算过程：

| 非终结符 | 产生式 | FIRST 集 |
|---------|--------|---------|
| F → (E) \| id | 终结符开头 | FIRST(F) = { (, id } |
| T → FT' | F 不能推导出 ε | FIRST(T) = FIRST(F) = { (, id } |
| T' → *FT' \| ε | | FIRST(T') = { *, ε } |
| E → TE' | T 不能推导出 ε | FIRST(E) = FIRST(T) = { (, id } |
| E' → +TE' \| ε | | FIRST(E') = { +, ε } |

**最终结果：**
```
FIRST(E)  = { (, id }
FIRST(E') = { +, ε }
FIRST(T)  = { (, id }
FIRST(T') = { *, ε }
FIRST(F)  = { (, id }
```

## 十四、FOLLOW 集的求法

**定义：** FOLLOW(A) = { 在所有句型中，紧跟在非终结符 A 之后的终结符 }，若 A 是开始符号，则 `#`（输入结束标记）∈ FOLLOW(A)。

**计算规则（反复迭代直至不再变化）：**

| 规则编号 | 条件 | 操作 |
|---------|------|------|
| 规则1 | `A` 是开始符号 | 将 `#` 加入 FOLLOW(A) |
| 规则2 | 有产生式 `A → αBβ` | 将 FIRST(β) 中所有非 ε 元素加入 FOLLOW(B) |
| 规则3 | 有产生式 `A → αBβ`，且 ε ∈ FIRST(β) | 将 FOLLOW(A) 加入 FOLLOW(B) |
| 规则4 | 有产生式 `A → αB`（B 在最右） | 将 FOLLOW(A) 加入 FOLLOW(B) |

**口诀：**
- **紧跟后面的符号** → FIRST(β) 去掉 ε 加入 FOLLOW(B)
- **后面能推出 ε 或后面没东西** → 把左部的 FOLLOW 传给 B

**示例（同上文法）：**

**第一轮：**

| 步骤 | 产生式 | 操作 | 结果 |
|------|--------|------|------|
| 1 | E 是开始符号 | # → FOLLOW(E) | FOLLOW(E) = { # } |
| 2 | E → TE' | E' 在最右 → FOLLOW(E) → FOLLOW(E') | FOLLOW(E') = { # } |
| 3 | E → TE' | E' 紧跟 T，但 FIRST(E') = {+, ε} → + → FOLLOW(T) | FOLLOW(T) = { + } |
| 4 | E → TE'，ε ∈ FIRST(E') | FOLLOW(E) → FOLLOW(T) | FOLLOW(T) = { +, # } |
| 5 | E' → +TE' | E' 在最右 → FOLLOW(E') → FOLLOW(E') | 无变化 |
| 6 | E' → +TE' | T 后面跟 E'，FIRST(E') 含 + 和 ε → + → FOLLOW(T)，且 FOLLOW(E') → FOLLOW(T) | FOLLOW(T) = { +, # } |
| 7 | T → FT' | T' 在最右 → FOLLOW(T) → FOLLOW(T') | FOLLOW(T') = { +, # } |
| 8 | T → FT' | T' 紧跟 F，FIRST(T') = {*, ε} → * → FOLLOW(F) | FOLLOW(F) = { * } |
| 9 | T → FT'，ε ∈ FIRST(T') | FOLLOW(T) → FOLLOW(F) | FOLLOW(F) = { *, +, # } |
| 10 | T' → *FT' | T' 在最右 → FOLLOW(T') → FOLLOW(T') | 无变化 |
| 11 | T' → *FT' | F 后面是 T'，FIRST(T') = {*, ε} → * → FOLLOW(F)，且 FOLLOW(T') → FOLLOW(F) | 无变化 |
| 12 | F → (E) | E 后面是 ) → ) → FOLLOW(E) | FOLLOW(E) = { #, ) } |

**第二轮（传播 FOLLOW(E) 的新元素 ）：**

| 步骤 | 操作 | 结果 |
|------|------|------|
| 13 | FOLLOW(E) = { #, ) } → FOLLOW(E')（规则2，E→TE'） | FOLLOW(E') = { #, ) } |
| 14 | FOLLOW(E) = { #, ) } → FOLLOW(T)（E→TE'，ε ∈ FIRST(E')） | FOLLOW(T) = { +, #, ) } |
| 15 | FOLLOW(T) 更新 → FOLLOW(T')（规则4，T→FT'） | FOLLOW(T') = { +, #, ) } |
| 16 | FOLLOW(T) 更新 → FOLLOW(F)（T→FT'，ε ∈ FIRST(T')） | FOLLOW(F) = { *, +, #, ) } |

第三轮检查，无变化，结束。

**最终结果：**
```
FOLLOW(E)  = { #, ) }
FOLLOW(E') = { #, ) }
FOLLOW(T)  = { +, #, ) }
FOLLOW(T') = { +, #, ) }
FOLLOW(F)  = { *, +, #, ) }
```

---

## 十五、LL(1) 分析表的构造

**步骤：**

1. 计算所有非终结符的 FIRST 集和 FOLLOW 集
2. 对每个产生式 `A → α`，执行：
   - 对 FIRST(α) 中的每个终结符 `a`，填入 `M[A, a] = A → α`
   - 若 ε ∈ FIRST(α)，则对 FOLLOW(A) 中的每个终结符 `b`（含 `#`），填入 `M[A, b] = A → α`
3. 其余空白处为**语法错误**

**LL(1) 文法的判定条件：** 分析表中每个单元格最多只有一个产生式（无多重定义）。

**示例（同上文法）：**

| | id | + | * | ( | ) | # |
|------|------|------|------|------|------|------|
| **E** | E → TE' | | | E → TE' | | |
| **E'** | | E' → +TE' | | | E' → ε | E' → ε |
| **T** | T → FT' | | | T → FT' | | |
| **T'** | | T' → ε | T' → *FT' | | T' → ε | T' → ε |
| **F** | F → id | | | F → (E) | | |

表无冲突 → 该文法是 **LL(1) 文法**。

## 十六、消除左递归

**直接左递归消除：**

将 `A → Aα | β` 转换为：

```
A  → βA'
A' → αA' | ε
```

**示例：**
```
原： E → E+T | T
转换后：
  E  → TE'
  E' → +TE' | ε
```

**一般左递归消除（多产生式）：**

将 `A → Aα₁ | Aα₂ | ... | Aαₘ | β₁ | β₂ | ... | βₙ` 转换为：

```
A  → β₁A' | β₂A' | ... | βₙA'
A' → α₁A' | α₂A' | ... | αₘA' | ε
```

**间接左递归消除：**

需要先将间接左递归转换为直接左递归，再消除。

例如：`A → Bc`，`B → Ad` 构成间接左递归（A → Bc → Adc → ...）

步骤：排列非终结符为 A₁, A₂, ..., Aₙ，对每个 Aᵢ 的产生式中替换 Aⱼ（j < i），然后消除直接左递归。

## 十七、LR(0) 项目集规范族的构造

**步骤：**

1. **拓广文法**：添加 `S' → S`（S 为原开始符号）
2. **计算初始项目集**：I₀ = CLOSURE({ S' → ·S })
3. **构造 DFA**：
   - 对每个项目集 I，对每个文法符号 X，计算 GOTO(I, X) = CLOSURE({ A → αX·β | A → α·Xβ ∈ I })
   - 重复直到不再产生新项目集
4. 所有项目集的集合即为**项目集规范族**

**CLOSURE(I) 的计算：**
- I 中所有项目属于 CLOSURE(I)
- 若 `A → α·Bβ ∈ CLOSURE(I)` 且 `B → γ` 是产生式，则将 `B → ·γ` 加入 CLOSURE(I)
- 重复直到不再变化

**示例：**

文法：
```
(0) S' → S
(1) S → (S)S
(2) S → a
```

```
I₀: CLOSURE({S' → ·S})
    S' → ·S
    S  → ·(S)S      ← S 后面圆点，加入 S 的产生式
    S  → ·a

I₁: GOTO(I₀, S)
    S' → S·          ← 接受状态

I₂: GOTO(I₀, ()
    S  → (·S)S
    S  → ·(S)S       ← 圆点后是非终结符 S，加入 S 的产生式
    S  → ·a

I₃: GOTO(I₀, a)
    S  → a·

I₄: GOTO(I₂, S)
    S  → (S·)S

I₅: GOTO(I₂, ()      ← GOTO(I₂, () = 同 I₂
    （与 I₂ 相同，合并）

I₆: GOTO(I₂, a)      ← 同 I₃
    S  → a·

I₇: GOTO(I₄, ))
    S  → (S)·S
    S  → ·(S)S       ← 圆点后是 S
    S  → ·a

I₈: GOTO(I₇, S)
    S  → (S)S·

I₉: GOTO(I₇, ()       ← 同 I₂
I₁₀: GOTO(I₇, a)      ← 同 I₃
```

**DFA 转移图：**
```
I₀ --S--> I₁
I₀ --(--> I₂
I₀ --a--> I₃
I₂ --S--> I₄
I₄ --)--> I₇
I₇ --S--> I₈
I₂ --(--> I₂（自环）
I₂ --a--> I₃
I₇ --(--> I₂
I₇ --a--> I₃
```

## 十八、SLR(1) 分析表的构造

**步骤：**

1. 构造 LR(0) 项目集规范族（上一节）
2. **ACTION 表**：
   - 若 `A → α·aβ ∈ Iᵢ` 且 `GOTO(Iᵢ, a) = Iⱼ`（a 是终结符）→ `ACTION[i, a] = sj`（移入状态 j）
   - 若 `S' → S· ∈ Iᵢ` → `ACTION[i, #] = acc`（接受）
   - 若 `A → α· ∈ Iᵢ`（归约项目），A ≠ S'，产生式编号为 k → 对 FOLLOW(A) 中每个 `a`，设 `ACTION[i, a] = rk`（用产生式 k 归约）
3. **GOTO 表**：
   - 若 `GOTO(Iᵢ, A) = Iⱼ`（A 是非终结符）→ `GOTO[i, A] = j`
4. 其余为**报错**

**SLR(1) 冲突检测：** 若上述填充过程中某单元格被多次定义且值不同，则存在冲突，文法不是 SLR(1) 文法。

## 十九、算符优先关系（FIRSTVT / LASTVT）的求法

**定义：**

| 集合 | 含义 |
|------|------|
| **FIRSTVT(A)** | A 能推导出的所有串中，第一个可能出现的终结符的集合 |
| **LASTVT(A)** | A 能推导出的所有串中，最后一个可能出现的终结符的集合 |

**FIRSTVT(A) 的计算规则：**

| 规则 | 条件 | 操作 |
|------|------|------|
| 规则1 | `A → a...` 或 `A → Ba...`（a 是终结符紧跟开头） | a 加入 FIRSTVT(A) |
| 规则2 | `A → B...`（B 是非终结符） | FIRSTVT(B) 加入 FIRSTVT(A) |

**LASTVT(A) 的计算规则：**

| 规则 | 条件 | 操作 |
|------|------|------|
| 规则1 | `A → ...a` 或 `A → ...aB`（a 是终结符紧跟末尾） | a 加入 LASTVT(A) |
| 规则2 | `A → ...B`（B 是非终结符） | LASTVT(B) 加入 LASTVT(A) |

**优先关系的构造规则：**

| 关系 | 构造方法 |
|------|---------|
| **a = b** | 若存在产生式 `A → ...ab...`，则 a = b |
| **a < b** | 若存在产生式 `A → ...aB...`，且 b ∈ FIRSTVT(B)，则 a < b |
| **a > b** | 若存在产生式 `A → ...Bb...`，且 a ∈ LASTVT(B)，则 a > b |

**示例：**

文法：
```
E → E + T | T
T → T * F | F
F → ( E ) | id
```

**计算 FIRSTVT 和 LASTVT：**

| 非终结符 | FIRSTVT | LASTVT |
|---------|---------|--------|
| F | { (, id } | { ), id } |
| T | { *, (, id } | { *, ), id } |
| E | { +, *, (, id } | { +, *, ), id } |

**构造优先关系表：**

从产生式 `E → E + T` 可得：
- `+` 在 E+T 中：`+` 后跟 T，FIRSTVT(T) 中的元素满足 `+ < *`，`+ < (`，`+ < id`
- `+` 在 E+T 中：`+` 前面是 E，LASTVT(E) 中的元素满足 `+ > +`，`* > +`，`) > +`，`id > +`

从产生式 `F → ( E )` 可得：`(` 和 `)` 相邻 → `( = )`

| | + | * | ( | ) | id | # |
|------|------|------|------|------|------|------|
| **+** | > | < | < | > | < | > |
| *** | > | > | < | > | < | > |
| **(** | < | < | < | = | < | |
| **)** | > | > | | > | | > |
| **id** | > | > | | > | | > |
| **#** | < | < | < | | < | |

无冲突 → 该文法是 **算符优先文法 (OPG)**。

## 二十、最左素短语的求法（算符优先分析）

**算法步骤：**

在算符优先分析中，栈中存放的是符号序列。为了找到最左素短语：

1. 从栈顶向下找第一个满足 `sᵢ₋₁ < sᵢ` 的位置（即优先级关系从小到大）
2. 从该位置继续向下找，直到遇到 `sⱼ > sⱼ₊₁`（优先级关系从大到小）
3. `sᵢ...sⱼ` 之间的子串就是**最左素短语**，进行归约

**示例：**

分析串 `id + id * id`，栈和输入的变化：

```
栈              输入          动作
#               id+id*id#     
# id            +id*id#       归约 id → F
# F             +id*id#       归约 F → T → E
# T             +id*id#       
# T +           id*id#        移入 +
# T + id        *id#          归约 id → F
# T + F         *id#          归约 F → T
# T + T         *id#          
# T + T *       id#           移入 *
# T + T * id    #             归约 id → F
# T + T * F     #             归约 T*F → T
# T + T         #             归约 E+T → E
# E             #             接受
```

## 二十一、语法制导翻译（SDT）示例

**综合属性（S-属性）示例：**

文法带语义规则：

| 产生式 | 语义规则 |
|--------|---------|
| `L → E = n` | `L.val = E.val` |
| `E → E₁ + T` | `E.val = E₁.val + T.val` |
| `E → T` | `E.val = T.val` |
| `T → T₁ * F` | `T.val = T₁.val * F.val` |
| `T → F` | `T.val = F.val` |
| `F → ( E )` | `F.val = E.val` |
| `F → num` | `F.val = num.lexval` |

输入 `3 + 4 * 5`，计算过程：

```
        E.val = 23
       / | \
      E  +  T
     |      |
     T      T * F → T.val = 20
     |      |    |
     F      F    F
     |      |    |
     3      4    5

自底向上计算：
F(3).val = 3 → T.val = 3 → E.val = 3
F(4).val = 4 → T.val = 4
F(5).val = 5
T(4)*F(5) → T.val = 20
E(3) + T(20) → E.val = 23
```

> **总结关系图**：
>
> 文法 → 推导（最左/最右） → 句型 → 短语/句柄
>
> 自顶向下 = 最左推导 = LL 系列 = FIRST/FOLLOW = 预测分析表
>
> 自底向上 = 最右推导逆过程 = LR 系列 = 项目集规范族 = ACTION/GOTO 表
>
> 算符优先 = 特殊自底向上 = 优先关系表 = 最左素短语
>
> 表驱动 = 统一的分析框架 = 预构造分析表 + 驱动程序
