# 编译原理核心术语详解

## 一、基本概念

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **文法 (Grammar)** | Grammar | 形式化描述语言结构的规则系统，由终结符、非终结符、产生式和开始符号组成 | 是所有分析方法的理论基础 |
| **终结符 (Terminal)** | Terminal Symbol | 构成语言基本符号的不可再分的元素，如关键字、运算符、标识符 | 是句型中最终出现的符号 |
| **非终结符 (Non-terminal)** | Non-terminal Symbol | 表示语法结构的语法变量，可被进一步展开 | 通过产生式推导为终结符串 |
| **产生式 (Production)** | Production Rule | 形如 A → α 的规则，定义非终结符如何推导 | 文法的核心组成部分 |
| **开始符号 (Start Symbol)** | Start Symbol | 文法中特殊的非终结符 S，所有合法句子都从它推导出来 | 推导的起点 |

## 二、推导与句型

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **推导 (Derivation)** | Derivation | 从开始符号出发，反复应用产生式替换非终结符，得到句子的过程 | 分为最左推导和最右推导 |
| **最左推导 (Leftmost Derivation)** | Leftmost Derivation | 每步都替换最左边的非终结符 | 与自顶向下分析对应 |
| **最右推导 (Rightmost Derivation)** | Rightmost Derivation | 每步都替换最右边的非终结符，又称规范推导 | 与自底向上分析（规范规约）对应 |
| **句型 (Sentential Form)** | Sentential Form | 从开始符号经过若干步推导得到的符号串（可含非终结符） | 句子是只含终结符的特殊句型 |
| **句子 (Sentence)** | Sentence | 只含终结符的句型，是语言中的合法串 | 是推导的最终结果 |
| **短语 (Phrase)** | Phrase | 句型中由某个非终结符推导出的子串。若 S ⇒* αAδ 且 A ⇒+ β，则 β 是句型 αβδ 中相对于 A 的短语 | 是规范规约中识别归约对象的基础 |
| **直接短语 (Immediate Phrase)** | Simple Phrase | 一步推导得到的短语，即 A → β 中 β 是相对于 A 的直接短语 | 句柄是最左直接短语 |
| **句柄 (Handle)** | Handle | 句型的最左直接短语，是自底向上分析中当前应被归约的子串 | 规范归约的核心概念，LR 分析器寻找句柄 |
| **规范句型 (Canonical Form)** | Canonical Sentential Form | 由最右推导得到的句型 | 规范规约的操作对象 |

## 三、语法分析方法总览

| 术语 | 英文 | 含义 | 代表算法 | 与其他概念的联系 |
|------|------|------|---------|-----------------|
| **自顶向下分析 (Top-down)** | Top-down Parsing | 从开始符号出发，逐步推导出输入串（构造语法树从根到叶） | LL(1)、递归下降、预测分析 | 对应最左推导；需要消除左递归和提取左公因子 |
| **自底向上分析 (Bottom-up)** | Bottom-up Parsing | 从输入串出发，逐步归约到开始符号（构造语法树从叶到根） | LR(0)、SLR(1)、LALR(1)、LR(1)、算符优先 | 对应最右推导的逆过程（规范规约） |
| **预测分析 (Predictive Parsing)** | Predictive Parsing | 根据向前看符号（lookahead）选择产生式，无需回溯 | LL(1) 分析表 | 是自顶向下分析的高效实现方式 |
| **递归下降 (Recursive Descent)** | Recursive Descent | 每个非终结符对应一个递归函数来实现分析 | 手写编译器常用 | 是预测分析的一种实现形式 |

## 四、LL 分析相关

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **LL(k) 文法** | LL(k) Grammar | 从左到右扫描、最左推导、向前看 k 个符号即可唯一确定产生式的文法 | LL(1) 是最常用的情况 |
| **FIRST 集** | FIRST Set | 从某个符号串能推导出的所有可能的第一个终结符的集合 | 用于构造 LL(1) 分析表 |
| **FOLLOW 集** | FOLLOW Set | 在某个句型中，紧跟在某个非终结符之后可能出现的所有终结符的集合 | 与 FIRST 集一起构造 LL(1) 预测分析表 |
| **LL(1) 分析表 (Parsing Table)** | LL(1) Parse Table | 二维表，行是非终结符，列是终结符，表项是应选的产生式或报错 | 是表驱动的自顶向下分析的核心 |
| **左递归 (Left Recursion)** | Left Recursion | 形如 A → Aα 的产生式，会导致自顶向下分析无限循环 | 必须消除后才能使用 LL 分析 |
| **左公因子 (Left Factoring)** | Left Common Factor | 多个产生式有相同前缀（A → αβ₁ \| αβ₂），导致无法预测选择 | 提取左公因子后使文法满足 LL(1) 条件 |

## 五、LR 分析相关

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **LR(k) 文法** | LR(k) Grammar | 从左到右扫描、最右推导的逆过程、向前看 k 个符号即可唯一确定归约的文法 | LR(1) 能分析的文法类是 LL(1) 的超集 |
| **LR(0) 项目 (Item)** | LR(0) Item | 产生式中带一个圆点标记分析进度，如 A → α·β | 是构造 LR 分析状态的基础 |
| **LR(1) 项目** | LR(1) Item | LR(0) 项目加上向前看符号，如 [A → α·β, a] | 提供精确的向前看信息，避免冲突 |
| **项目集规范族 (Canonical Collection)** | Canonical Collection of Item Sets | 所有有效项目集的集合，每个项目集对应分析表的一个状态 | 是构造 LR 分析表的中间产物 |
| **GO 函数 (GOTO)** | GOTO Function | GOTO(I, X) 表示项目集 I 经符号 X 转移后得到的新项目集 | 构建分析状态之间的转移关系 |
| **CLOSURE 函数** | CLOSURE Function | 对项目集中圆点后的非终结符，将其所有产生式的初始项目加入集合 | 计算完整的项目集 |
| **SLR(1)** | Simple LR(1) | 用 FOLLOW 集来解决 LR(0) 中的冲突，比 LR(0) 强，比 LR(1) 弱 | 是 LR(0) 和 LR(1) 的折中 |
| **LALR(1)** | Lookahead LR(1) | 合并 LR(1) 项目集中同心（核心相同）的项目集以减少状态数 | Yacc/Bison 默认使用；能力介于 SLR(1) 和 LR(1) 之间 |

## 六、规范规约

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **规范规约 (Canonical Reduction)** | Canonical Reduction | 最右推导的逆过程，每步归约当前句型的最左句柄 | 是自底向上分析的正确归约策略 |
| **规范归约的步骤** | — | 1. 移入输入符号到栈中；2. 识别栈顶的句柄；3. 用产生式左部替换句柄 | LR 分析器通过状态栈自动完成此过程 |
| **活前缀 (Viable Prefix)** | Viable Prefix | 规范句型的前缀，该前缀不会越过句柄的右端 | LR 分析器栈中保存的内容就是活前缀 |

## 七、算符优先分析

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **算符优先文法 (Operator Precedence Grammar)** | OPG | 不含 ε 产生式，且任何两个终结符之间至多有一种优先关系的文法 | 是一种特殊的自底向上分析方法 |
| **算符优先关系 (Precedence Relation)** | Operator Precedence Relation | 终结符之间的三种关系：<（低于）、=（等于）、>（高于） | 用于确定归约的边界 |
| **优先关系表 (Precedence Table)** | Precedence Table | 存储所有终结符对之间优先关系的二维表 | 类似于 LR 分析表，但更简单 |
| **优先函数 (Precedence Function)** | Precedence Function | 用两个函数 f 和 g 代替优先关系表，将关系比较转化为数值比较 | 节省存储空间，f(a) < g(b) 等价于 a < b |
| **最左素短语 (Leftmost Prime Phrase)** | Leftmost Prime Phrase | 至少含一个终结符且不含更小的含终结符短语的短语 | 算符优先分析中的归约对象（不是句柄） |

### 算符优先 vs 规范规约

| 对比维度 | 算符优先分析 | 规范规约 (LR) |
|---------|------------|--------------|
| 归约对象 | 最左素短语 | 句柄 |
| 分析能力 | 较弱（仅 OPG 文法） | 强（LR 文法类） |
| 实现复杂度 | 简单 | 较复杂 |
| 错误检测 | 较弱 | 较强 |
| 典型应用 | 表达式解析 | 通用语法分析 |

## 八、表驱动分析

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **表驱动分析 (Table-driven Parsing)** | Table-driven Parsing | 使用预构造的分析表驱动分析动作（移入/归约/接受/报错），而非硬编码逻辑 | LL(1) 和 LR 系列都采用此模式 |
| **分析表 (Parse Table)** | Parse Table | 以当前状态和输入符号为索引，指示应执行的分析动作 | LR 分析表含 ACTION 和 GOTO 两部分 |
| **ACTION 表** | ACTION Table | LR 分析表的一部分，指示在当前状态和终结符下的动作（移入/归约/接受/报错） | 与 GOTO 表配合使用 |
| **GOTO 表** | GOTO Table | LR 分析表的一部分，指示在当前状态和非终结符下转移到哪个新状态 | 归约后使用 |
| **移入-归约冲突 (Shift-Reduce Conflict)** | Shift-Reduce Conflict | 在某个状态下，分析表同时指示移入和归约 | 说明文法不是该文法类（如不是 SLR），需要改写文法 |
| **归约-归约冲突 (Reduce-Reduce Conflict)** | Reduce-Reduce Conflict | 在某个状态下，分析表对同一输入有多个归约选择 | 文法有二义性或不适合该分析技术 |

## 九、冲突与二义性

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **二义文法 (Ambiguous Grammar)** | Ambiguous Grammar | 同一个句子存在多棵不同的语法树（多个最左推导） | 二义文法不是 LR 文法；可通过优先级/结合性规则解决 |
| **悬空 else (Dangling Else)** | Dangling Else | if-else 语句的经典二义性问题 | 通常通过"最近匹配"规则消除 |
| **优先级 (Precedence)** | Precedence | 运算符执行顺序的约定 | 可通过声明优先级消除移入-归约冲突 |
| **结合性 (Associativity)** | Associativity | 相同优先级运算符的结合方向（左结合/右结合） | 辅助消除冲突 |

## 十、文法类层次关系

```
文法能力（从弱到强）：

  LL(1)  ⊂  SLR(1)  ⊂  LALR(1)  ⊂  LR(1)

  算符优先文法 (OPG) 与上述系列有交集但不完全包含
```

| 文法类 | 分析方向 | 分析能力 | 分析表大小 | 实际应用 |
|-------|---------|---------|-----------|---------|
| LL(1) | 自顶向下 | 最弱 | 小 | 手写递归下降编译器 |
| SLR(1) | 自底向上 | 较强 | 中 | 教学示例 |
| LALR(1) | 自底向上 | 强 | 中 | Yacc / Bison / CUP |
| LR(1) | 自底向上 | 最强 | 大 | 理论研究，状态数过多 |
| OPG | 自底向上 | 特殊 | 小 | 表达式子语言 |

## 十一、中间代码与语义分析（扩展）

| 术语 | 英文 | 含义 | 与其他概念的联系 |
|------|------|------|-----------------|
| **属性文法 (Attribute Grammar)** | Attribute Grammar | 在上下文无关文法基础上附加语义属性和语义规则 | 语法分析的延伸，用于语义检查 |
| **S-属性定义 (S-attributed)** | S-attributed Definition | 只含综合属性的属性文法 | 可在自底向上分析中计算 |
| **L-属性定义 (L-attributed)** | L-attributed Definition | 属性依赖限于从左到右计算的属性文法 | 可在自顶向下分析中计算 |
| **语法制导翻译 (Syntax-Directed Translation)** | SDT | 在语法分析过程中嵌入语义动作 | 连接语法分析与中间代码生成 |
| **三地址码 (Three-Address Code)** | TAC | 中间代码形式，每条指令最多三个操作数 | 语法分析之后的中间表示 |
| **抽象语法树 (AST)** | Abstract Syntax Tree | 源代码语法结构的树形抽象表示 | 语法分析的输出，语义分析的输入 |

---

> **总结关系图**：
>
> 文法 → 推导（最左/最右） → 句型 → 短语/句柄
>
> 自顶向下 = 最左推导 = LL 系列 = FIRST/FOLLOW = 预测分析表
>
> 自底向上 = 最右推导逆过程 = LR 系列 = 项目集规范族 = ACTION/GOTO 表
>
> 算符优先 = 特殊自底向上 = 优先关系表 = 最左素短语
>
> 表驱动 = 统一的分析框架 = 预构造分析表 + 驱动程序
